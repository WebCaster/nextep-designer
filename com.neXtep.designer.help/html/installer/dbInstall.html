Installing the neXtep installer


The installer is a Java 1.4-based program which is able to deploy deliveries generated with neXtep Designer. The installer is free and you don't need any specific license so feel free to install it wherever you need. Typically, you will need an installer on each target database server where you need to deploy neXtep deliveries.



Download / system settings


These steps will guide you through the neXtep installer installation on a target machine. Note that those steps will be the same each time you install the neXtep installer on a target machine.


  • Start by downloading the latest release of the neXtep installer

  • Put the archive on the server you would like to use as the database server of the repository.

  • Unzip the installer in a directory you choose. We will call this directory $INSTALLER_DIR

  • Define a new global system variable NXTP_HOME which points to $INSTALLER_DIR. Add this variable to your global path.

Linux / Unix

Windows

export NXTP_HOME=/opt/neXtep/installer (Assuming /opt/neXtep/installer is your “$INSTALLER_DIR”)

export PATH=$PATH:$NXTP_HOME

chmod +x $NXTP_HOME/nxtp.sh

And add the 2 export lines to the .bashrc file of the home directory of system users which will use the installer for convenience.

set NXTP_HOME=c:\neXtep\installer

set PATH=%PATH%;%NXTP_HOME%

and / or right click on “Computer” > properties

Click on advanced system settings

Click on Environment variables

Add the NXTP_HOME variable

Modify the PATH variable


  • Edit the file $INSTALLER_DIR/properties/neXtep.properties and adjust the settings to define the database schema which will be used as the administration schema for neXtep deployments data. This is a very small schema used by the installer to keep track of what has been deployed where and containing some deployment logs.

  • Check that you have a proper java 1.4 or higher on your machine : java -version


Initialization

Everything is ready for the installer initialization :

  • Open a shell window on the target machine

  • Execute the command “nxtp -install” (Windows) or “nxtp.sh -install” (Linux / Unix)

  • The administration database schema is now configured


Delivery deployment

  • Go to the directory which contains the neXtep delivery to install

  • Execute the command “nxtp” (Windows) or “nxtp.sh” (Linux)

  • Enter the user / password / database id of the schema into which you would like to install the repository (or the schema of a currently installed repository for repository upgrades)

  • The repository should be installed without errors on your system.


Your delivery is installed. Note that the user / password / database information you provided in the installation should be the same each time you deploy a package to this schema.

You can use a non-interactive deployment by the following command line :

nxtp.sh -user=username -pass=password -db=databasename
