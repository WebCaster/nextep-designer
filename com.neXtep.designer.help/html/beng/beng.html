The delivery engine

neXtep Designer comes with a great feature : the delivery engine. This engine handles the “glue” between version information and database structural upgrade deliveries. In this section we will see the features offered by the product to help you delivering your database versions and automate the upgrade process.



Database deliveries considerations


Delivering updates for an existing production database has specific constraints due to the persistent aspect of a database. Unlike a Web or standalone application, you cannot remove and replace your previous release by the new one as this operation would loose all your persistent data. Instead you need to provide incremental scripts which literally upgrade your previous release to the new one.


This could be very simple for small projects, but as your project grows, you will need to maintain several releases of your database schema in parallel :

  • The production release which you might need to patch

  • The tested release which you also might need to patch

  • Your development release

  • For large projects you might also have several releases being developed and tested in parallel


But the incremental aspect of database deliveries is not fitting well with the development parallelism... This is why neXtep designer provides features for managing and deploying these deliveries : the delivery engine.


Database release delivery


The delivery engine intensively uses information from the Version Control System to keep track of all changes you make to your schema objects and automates the generation of the needed incremental scripts. More, for multi-team projects you will be able to define dependencies in your deliveries which will make the delivery engine embed external-team deliveries into your development.


The neXtep installer


A free installer is provided with neXtep designer which will be able to deploy your generated deliveries to any neXtep-managed database or to convert any existing database to a neXtep-managed database.
