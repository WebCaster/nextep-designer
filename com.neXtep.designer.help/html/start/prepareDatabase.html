Preparing the repository database

neXtep Designer needs a repository to store its data and cannot run without. The deployment of the repository database is fully automatized through your neXtep Designer client except for the database schema/user creation.


Repository database location


We advise you to start with a database server located on your local network for the response times to be optimal. We advise you to create a dedicated user for the neXtep repository for security purposes.



Repository database vendor


NeXtep designer currently supports 2 database vendors :

  • Oracle

  • MySQL


Releases of neXtep designer are heavily tested against Oracle 9.2 or later and MySQL 5.0 or later. Note that the product may work well with previous releases of the database engines but we haven't tested it.


Oracle repository initialization


Well, if your are a DBA, you don't need us : create a new user of your choice which can connect and can at least create / drop / alter tables and views and manipulate data.


For others, either ask your DBA or create the following default user.

MySQL repository initialization


Well, if your are a DBA, you don't need us : create a new user of your choice which can connect and can at least create / drop / alter tables and views and manipulate data. If the database is not located on the machine on which neXtep Designer runs, please make sure the user can be remotely accessed.


For others, either ask your DBA or create the following default user.

You need your mysql root user password :

>mysql -uroot -p

Enter password:

Welcome to the MySQL monitor. Commands end with ; or \g.

Your MySQL connection id is 59

Server version: 5.0.67-community-nt MySQL Community Edition (GPL)


Type 'help;' or '\h' for help. Type '\c' to clear the buffer.


mysql> create user nextep_user identified by "nextep_user";

Query OK, 0 rows affected (0.20 sec)


mysql> create database nextep_repos;

Query OK, 1 row affected (0.00 sec)


mysql> grant all on nextep_repos.* to nextep_user;

Query OK, 0 rows affected (0.00 sec)


mysql> grant all on nextep_repos.* to nextep_user@'%';

Query OK, 0 rows affected (0.00 sec)


That's it, you now have your repository user. Go on with the next section.
