Installing / upgrading the repository database


neXtep designer needs a repository database to store versioning information about all the objects of the environment. This repository database needs to be created before the very first start of the neXtep designer environment.

The repository database is generated automatically when you start neXtep designer for the first time.


You only need to have previously created a database user for this repository schema. Indicate the repository connection where you want the repository to be generated and neXtep designer will ask you to confirm repository creation.


Repository upgrades


NeXtep designer upgrades very frequently to keep you up to date with the latest bug fixes and new features.

Some updates need to alter your repository. In this case, neXtep designer will prompt you to upgrade your repository when you restart the environment after a software update.

When you click OK, you will see the installer GUI which will deploy the updates. After this phase, you can continue to use neXtep designer as before.


Note that if you work with other people on the same repository, the upgrade need only to be performed once (by the first person who updates his client).
