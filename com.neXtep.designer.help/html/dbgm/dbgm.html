The database model

The database model is the module which allows neXtep designer to provide database design features.

In this section, we tried to put maximum information in the common section. All specificities of vendors will be documented in the vendor-specific section.



The database common model


First, there is the database common model which is an abstraction of database models regardless of the vendor. It contains all basic / generic elements of database development such as tables, keys, indexes, views, triggers.


The database Oracle model


The database Oracle model extends the common model to add Oracle-specific elements such as tablespaces, partitioning, clustering, PL/SQL, compiler.


The database MySQL model


The database MySQL model extends the common model to add MySQL-specific elements such as MySQL datatypes (enum, etc), auto-incremented columns, signed / unsigned datatypes...


More to come...


Several other vendors are currently being developed by the neXtep development team to support DB2, Sybase and PostGreSQL.
