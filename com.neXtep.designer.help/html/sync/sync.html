The synchronization engine

When you work with neXtep designer, you develop your changes within the neXtep repository rather than directly on your development database. Once you have finished a development task, you will need to publish this work to the development database : this is the job of the synchronization engine.


The synchronization is a comparison between an existing database structure and its repository-version allowing you to publish repository changes to the development database or to integrate database changes into the repository view.


The following sections of this chapter will introduce you how to (reverse) synchronize the repository to / from an existing database.
