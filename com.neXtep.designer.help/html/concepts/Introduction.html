Introduction

neXtep designer introduces a new way of designing databases and allows you to control the lifecycle of your database developments.



The repository


The main concept to understand is that when you work in the neXtep environment, you make your changes in a version repository instead of altering your development database directly. The version repository logs all changes made to “objects” of your database and creates versions for these modifications.


A version is a “snapshot” of an object structure (table, index, key, stored package, diagram,...) at a given time.


During development phase, you will be able to synchronize modifications made to your disconnected repository with a development database very frequently (when you say so).


When you need to deliver your developments to another environment / team (integration, production), neXtep designer will be able to compute all changes made to your database structure since your last delivery (or since any given version) and will generate all SQL statements which will migrate the database to your new structure.



Repository versioning


Versioning is the core of neXtep designer: any single element of the neXtep designer environment has version information. Therefore, any element has a lifecycle and any change made to any element can be tracked.



The ecosystem


NeXtep designer is an ecosystem of tools which, altogether, constitute the whole neXtep designer environment. This documentation will be organized by the same frontiers as the different modules of neXtep designer, to allow proper maintenance.


Here are the modules of neXtep designer :

  • Core : provides a common base for all other modules.

  • Version Control System : provides all versioning features to track changes of any element of the environment.

  • Database generic model : provides a common shared layer for representing / modeling a database structure, whatever the database vendor.

  • Database Oracle model : provides a layer for modeling Oracle-specific database structure

  • Database MySQL model : provides a layer for modeling MySQL-specific database structure

  • SQL generator : provides all functionalities to generate SQL-statements from database structural changes.

  • Build engine : provides all functionalities to build exportable deployment packages.
